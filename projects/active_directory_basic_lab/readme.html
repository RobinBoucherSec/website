üß© Overview

This lab provides a simplified environment to explore the core components and functionalities of Microsoft Active Directory using virtual machines. It includes domain controller setup, user and group management, Group Policy Objects (GPOs), DNS, and more.

The lab simulates a small enterprise network with a single domain: lab.local.

üéØ Lab Objectives

By the end of this lab, you will be able to:

  • Install and configure a Domain Controller
  • Manage users, groups, and computers
  • Apply Group Policy settings
  • Understand DNS integration with AD
  • Explore Organizational Units (OUs) and delegation
  • Practice basic AD security configurations
  • Troubleshoot common AD issues

üõ† Requirements

Hardware/Software:

  • A computer running Windows 10/11 Pro or Server OS
  • Hyper-V or VMware Workstation Player / VirtualBox
  • At least 8GB RAM (16GB recommended)
  • 50GB free disk space

VMs Required:

Machine OS Role
DC01 Windows Server 2019/2022 Domain Controller
CLIENT01 Windows 10/11 Domain Member
(Optional) SQL01 Windows Server Additional Service

üß∞ Lab Setup Instructions

  1. Download ISOs
    • Windows Server 2019/2022
    • Windows 10/11
  2. Create Virtual Machines
    • Set up DC01 with at least 2GB RAM and 40GB HDD.
    • Set up CLIENT01 with Windows 10/11.
  3. Network Configuration
    • Use an Internal Network or Private Network switch.
    • Assign static IP addresses or use DHCP after promoting DC.
  4. Promote DC01 to Domain Controller
    • Install AD DS role via PowerShell or Server Manager.
    • Promote server to domain controller (dcpromo or PowerShell).
    • Create a new forest: lab.local
  5. Join CLIENT01 to the Domain
    • Rename machine and join domain lab.local
    • Reboot and verify domain membership.
  6. Verify DNS
    • Ensure DNS is integrated with AD.
    • Test name resolution from client.

üß™ Lab Tasks & Exercises

‚úÖ Task 1: Active Directory Installation

  • Install AD DS on DC01
  • Promote server to DC for lab.local

‚úÖ Task 2: User & Group Management

  • Create OUs: Users, Groups, Computers
  • Add 5 test users (e.g., Alice, Bob, etc.)
  • Create groups: IT, Sales, HR
  • Add users to appropriate groups

‚úÖ Task 3: Group Policy Configuration

  • Create GPO to disable command prompt
  • Create GPO to set wallpaper and screen timeout
  • Link GPOs to OUs

‚úÖ Task 4: Computer Management

  • Join CLIENT01 to the domain
  • Move computer object to Computers OU
  • Verify GPO application using gpresult

‚úÖ Task 5: Delegation & Permissions

  • Delegate control of Users OU to a non-admin user
  • Allow password reset and group membership changes

‚úÖ Task 6: DNS Management

  • Add forward lookup zone if not created automatically
  • Create A records and CNAME records
  • Test DNS resolution from client

‚úÖ Task 7: Backup & Restore (Optional)

  • Backup system state of DC
  • Simulate accidental deletion and restore using AD Recycle Bin or backup

üõ° Tips & Best Practices

  • Always use strong passwords in a lab that may be exposed.
  • Keep snapshots of VMs before major changes.
  • Use descriptive names for GPOs and OUs.
  • Document your configuration steps.
  • Regularly review event logs for troubleshooting.

‚ùì Troubleshooting

Issue Solution
Client cannot resolve domain Check DNS settings; ensure pointing to DC
Cannot join domain Verify network connectivity, firewall rules, and DNS
GPO not applying Run gpupdate /force, check permissions, and links
Replication errors Wait for replication or force via repadmin
Missing roles/features Use Add-WindowsFeature or Server Manager
